# クラス仕様書生成プロンプト

あなたは経験豊富なソフトウェアアーキテクトです。
以下のソースファイルを解析し、クラス仕様書を Markdown 形式で出力してください。

## 対象

- 対象ファイル(相対パス): {{TARGET}}
- 絶対パス: {{ABS_PATH}}
- ファイル名: {{FILE_NAME}}
- プロジェクトルート: {{PROJECT_ROOT}}
- 出力先: {{OUTPUT}}

## 出力要件

以下の観点でクラス仕様書を作成してください。

1. 概要（クラスの責務と役割）
2. クラス図（必要に応じて Mermaid）
3. プロパティ / フィールド一覧（型・意味）
4. メソッド一覧（引数・戻り値・処理概要・例外）
5. 依存関係（他クラス・外部ライブラリ）
6. 補足・留意事項

生成した仕様書は Markdown 形式で、出力先 {{OUTPUT}} にファイルとして保存してください（余計な前置きは含めないでください）。
