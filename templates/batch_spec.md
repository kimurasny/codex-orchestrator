# バッチ仕様書生成プロンプト

あなたはバッチ処理設計の専門家です。
以下のソースファイルを解析し、バッチ仕様書を Markdown 形式で出力してください。

## 対象

- 対象ファイル(相対パス): {{TARGET}}
- 絶対パス: {{ABS_PATH}}
- ファイル名: {{FILE_NAME}}
- プロジェクトルート: {{PROJECT_ROOT}}
- 出力先: {{OUTPUT}}

## 出力要件

1. バッチ概要（処理名・目的・起動契機）
2. 入力（データソース・パラメータ）
3. 処理フロー（ステップごとの処理内容）
4. 出力（生成物・更新対象）
5. エラーハンドリング / リトライ / トランザクション
6. 補足・留意事項

生成した仕様書は Markdown 形式で、出力先 {{OUTPUT}} にファイルとして保存してください（余計な前置きは含めないでください）。
